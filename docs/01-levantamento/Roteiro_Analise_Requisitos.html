4. Requisitos Nao-Funcionais
Nao sao sobre "o que fazer", mas sobre "como o sistema deve ser".

Desempenho

Requisito:
Em caso de consultas longas, adicionar tela de carregamento. Paginacao nas listagens de turmas e reservas.
Como medir / Metrica:
Tempo de resposta maior que 2s exibe indicador de carregamento. Consultas ao banco otimizadas com indices.

Seguranca

Requisito:
DPO, criptografia. Protecao contra SQL injection e XSS. Controle de acesso por perfil (Supervisao/Suporte/Docente).
Como medir / Metrica:
Hash de senha (password_hash/bcrypt) e conexao segura. PDO com prepared statements. Sessoes com token CSRF.

Disponibilidade

Requisito:
Autofuncionamento. O sistema deve estar acessivel durante o horario de expediente (seg a sab, 7h as 22h).
Como medir / Metrica:
O sistema funciona sem interrupcoes durante o expediente. Hospedado em servidor local (XAMPP/Laragon) ou servidor da unidade.

Usabilidade

Requisito:
Visualizacao geral de reservas (tabela ou algo parecido). Interface semelhante a planilha atual para facilitar a transicao da equipe.
Como medir / Metrica:
O usuario nao tem dificuldades no uso. Treinamento maximo de 1 hora para a equipe. Layout com cores do SENAC (#004C94 azul, #F7941D laranja).

Escalabilidade

Requisito:
O sistema deve crescer sem problemas de desempenho. Se aprovado, podera ser usado em outras unidades do SENAC Minas.
Como medir / Metrica:
Aceita concorrencia (acessos simultaneos) sem problemas. Banco de dados suporta multiplas unidades (tabela unidade com FK em todas as entidades). Constraint UNIQUE garante integridade mesmo com acessos simultaneos.
5. Cenario de Excecao
Escolha uma situacao critica e descreva o comportamento esperado do sistema.
Cenario escolhido: Conflito de agendamento (reservas)

O que aconteceu?

O usuario (Suporte) tenta reservar a Sala 1 para a turma 006.2026.0012 (Logistica) no turno da Manha do dia 30/06/2026, mas a sala ja esta reservada para a turma 006.2026.0009 (APQ Admin) nesse mesmo horario.

Quem foi afetado?

Usuario Suporte que tentou criar a reserva. Indiretamente, a turma de Logistica que precisa de uma sala.

Qual a acao esperada do sistema?

Bloquear a reserva duplicada (constraint UNIQUE no banco). Exibir alerta vermelho: "Conflito detectado! Sala 1 ja reservada para turma 006.2026.0009 neste horario." Sugerir salas disponiveis no mesmo turno: "Salas disponiveis: Sala 3, Sala 5, Lab 2".

Qual mensagem o usuario deve ver?

"Conflito detectado! A Sala 1 ja esta reservada para a Turma 006.2026.0009 (APQ Serv. Administrativos) no turno da Manha em 30/06/2026. Salas disponiveis neste horario: Sala 3, Sala 5, Sala 8, Lab 2."

O que acontece com os dados?

A reserva conflitante NAO e salva no banco de dados. A constraint UNIQUE (id_sala, data, turno) impede fisicamente a insercao duplicada. Nenhum dado existente e alterado. O usuario pode escolher outra sala ou outro horario e tentar novamente. A tentativa de conflito pode ser registrada no log_alteracao para auditoria.

5b. Cenarios de Excecao Adicionais
Outros cenarios marcados no checklist acima, detalhados para complementar a analise.
Dado invalido / inconsistente

O que aconteceu?

O usuario tenta cadastrar uma turma com codigo duplicado (006.2026.0009 ja existe) ou com data de termino anterior a data de inicio.

Acao do sistema

Bloquear o cadastro e exibir mensagem: "Codigo de turma ja cadastrado" ou "Data de termino deve ser posterior a data de inicio". Dados nao sao salvos.

Tentativa de invasao / fraude

O que aconteceu?

Tentativa de SQL injection via campo de login ou tentativa de acesso a pagina de admin sem permissao.

Acao do sistema

PDO com prepared statements neutraliza SQL injection automaticamente. Acesso negado para paginas sem permissao com redirect para o painel do perfil. Tentativa registrada no log_alteracao.

Timeout / lentidao extrema

O que aconteceu?

Consulta ao calendario anual com muitas reservas demora mais de 2 segundos para carregar.

Acao do sistema

Exibir indicador de carregamento (spinner) para o usuario. Paginar resultados. Usar cache para consultas frequentes. Se o tempo exceder 30s, exibir: "A consulta demorou mais que o esperado. Tente filtrar por um periodo menor."

Indisponibilidade de servico externo

O que aconteceu?

O banco de dados MySQL esta fora do ar ou o servidor Apache parou de responder.

Acao do sistema

Exibir pagina de erro amigavel: "Sistema temporariamente indisponivel. Contate o suporte tecnico." Nao exibir mensagens de erro tecnicas (stack trace, SQL) para o usuario final. Registrar erro em arquivo de log do servidor.
