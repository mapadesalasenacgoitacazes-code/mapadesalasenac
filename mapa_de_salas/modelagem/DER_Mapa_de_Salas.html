Relacionamentos

UNIDADE 1 : N SALA — uma unidade tem varias salas
UNIDADE 1 : N FERIADO — cada unidade define seus feriados
UNIDADE 1 : N USUARIO — usuarios pertencem a uma unidade
SALA 1 : N RECURSO_SALA — sala tem varios recursos
SALA 1 : N RESERVA — sala recebe varias reservas
CURSO 1 : N TURMA — um curso tem varias turmas
TURMA 1 : N RESERVA — turma ocupa sala em varios dias
TURMA 1 : N DIA_CANCELADO — turma pode ter dias cancelados
USUARIO 1 : N RESERVA — quem criou a reserva
USUARIO 1 : N LOG_ALTERACAO — rastreia acoes do usuario
TURMA N : N USUARIO — via PROFESSOR_TURMA (varios professores por turma)
Constraint critica (RN02): A tabela RESERVA tem UNIQUE(id_sala, data, turno), que impede fisicamente no banco de dados que duas turmas sejam alocadas na mesma sala, no mesmo dia, no mesmo turno. Essa e a regra de negocio mais importante do sistema.

Views: O schema inclui 4 views prontas: vw_progresso_turma (calcula progresso automatico), vw_historico_salas_turma (por quais salas cada turma passou), vw_ocupacao_hoje (status de todas as salas no dia atual) e vw_professores_turma (lista professores por turma).
